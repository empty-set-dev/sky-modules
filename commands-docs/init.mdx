# sky init

Initialize and configure various aspects of a Sky Modules project.

## Description

The `init` command helps set up and configure different parts of your Sky Modules project, from basic configuration files to development environment settings.

## Usage

```bash
sky init [command] [options]
```

## Available Subcommands

### sky-config

Initialize the Sky configuration file.

```bash
sky init sky-config
```

Creates `.sky/sky.config.ts` with default project configuration.

### package.json

Update the project's package.json file with Sky Modules dependencies and scripts.

```bash
sky init package.json
```

### ts-configs

Update TypeScript configuration files for all modules, examples, and apps.

```bash
sky init ts-configs
```

Ensures all TypeScript configurations are properly set up across the project.

### packages

Initialize packages and configurations.

```bash
sky init packages
```

Sets up necessary packages and their configurations.

### .gitignore

Initialize or update the .gitignore file.

```bash
sky init .gitignore
```

Creates a .gitignore file if it doesn't exist, with appropriate patterns for Sky Modules projects.

### vscode-workspace

Update VSCode workspace configuration in the project root.

```bash
sky init vscode-workspace
```

Sets up VSCode workspace settings for optimal development experience.

### vscode-workspace-tasks

Update VSCode workspace tasks for a specific app.

```bash
sky init vscode-workspace-tasks <app-name>
```

**Arguments:**
- `app-name` (required) - Name of the Sky app

**Example:**
```bash
sky init vscode-workspace-tasks my-app
```

## Default Behavior

Running `sky init` without any subcommand will initialize all configurations:

```bash
sky init
```

This is equivalent to running all initialization subcommands in sequence.

## Examples

```bash
# Initialize everything
sky init

# Initialize only Sky configuration
sky init sky-config

# Update package.json
sky init package.json

# Set up TypeScript configurations
sky init ts-configs

# Initialize VSCode workspace for app
sky init vscode-workspace-tasks my-app
```

## Notes

- Initialization commands are safe to run multiple times
- Existing configurations will be updated rather than overwritten
- The default behavior ensures a complete project setup
