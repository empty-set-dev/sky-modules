# sky web

Управление веб-приложениями с использованием фреймворка Vike.

## Описание

Команда `web` предоставляет полный набор инструментов для разработки, сборки и развертывания веб-приложений с использованием фреймворка Vike в проектах Sky Modules.

## Использование

```bash
sky web <команда> [опции]
```

## Доступные подкоманды

### init

Инициализация нового веб-приложения.

```bash
sky web init <имя-приложения> [опции]
```

**Аргументы:**
- `имя-приложения` (обязательный) - Имя Sky приложения для инициализации

**Опции:**
- `--server` - Инициализировать только серверное приложение (boolean)

**Пример:**
```bash
sky web init my-web-app
sky web init my-server-app --server
```

### dev

Запуск сервера разработки с горячей перезагрузкой.

```bash
sky web dev <имя-приложения> [опции]
```

**Аргументы:**
- `имя-приложения` (обязательный) - Имя Sky приложения для запуска

**Опции:**
- `--port` - Номер порта для сервера разработки (по умолчанию: 3000)
- `--open` - Открыть в браузере после запуска (по умолчанию: false)
- `--host` - Открыть доступ к серверу из сети (по умолчанию: false)

**Пример:**
```bash
sky web dev my-app
sky web dev my-app --port 4000 --open
sky web dev my-app --host --port 3000
```

### build

Сборка веб-приложения для продакшена.

```bash
sky web build <имя-приложения>
```

**Аргументы:**
- `имя-приложения` (обязательный) - Имя Sky приложения для сборки

**Пример:**
```bash
sky web build my-app
```

### preview

Предварительный просмотр продакшен-сборки локально.

```bash
sky web preview <имя-приложения> [опции]
```

**Аргументы:**
- `имя-приложения` (обязательный) - Имя Sky приложения для просмотра

**Опции:**
- `--port` - Номер порта для сервера предпросмотра (по умолчанию: 3000)
- `--open` - Открыть в браузере после запуска (по умолчанию: false)
- `--host` - Открыть доступ к серверу из сети (по умолчанию: false)

**Пример:**
```bash
sky web preview my-app
sky web preview my-app --port 5000 --open
```

### start

Запуск продакшен-сервера.

```bash
sky web start <имя-приложения> [опции]
```

**Аргументы:**
- `имя-приложения` (обязательный) - Имя Sky приложения для запуска

**Опции:**
- `--port` - Номер порта для продакшен-сервера (по умолчанию: 3000)
- `--open` - Открыть в браузере после запуска (по умолчанию: true)
- `--host` - Открыть доступ к серверу из сети (по умолчанию: false)

**Пример:**
```bash
sky web start my-app
sky web start my-app --port 8080
sky web start my-app --host --no-open
```

## Пример рабочего процесса

```bash
# Инициализировать новое веб-приложение
sky web init my-awesome-app

# Запустить сервер разработки
sky web dev my-awesome-app --open

# Собрать для продакшена
sky web build my-awesome-app

# Предпросмотр продакшен-сборки
sky web preview my-awesome-app

# Запустить продакшен-сервер
sky web start my-awesome-app
```

## Примечания

- Сервер разработки включает горячую замену модулей (HMR)
- Продакшен-сборки оптимизированы для производительности
- Опция `--host` позволяет получить доступ с других устройств в сети
- Консоль автоматически очищается при запуске команд dev, preview или start
