# sky test

Запуск тестов с использованием фреймворка Jest.

## Описание

Команда `test` предоставляет удобный способ запуска тестов в вашем проекте Sky Modules с использованием Jest. Она поддерживает тестирование конкретных папок или запуск всех тестов в проекте.

## Использование

```bash
sky test [папка]
```

## Аргументы

- `папка` (опциональный) - Путь к конкретной папке для тестирования. Если не указан, запускаются все тесты.

## Примеры

```bash
# Запустить все тесты в проекте
sky test

# Протестировать конкретный модуль
sky test modules/auth

# Протестировать конкретную директорию
sky test src/components

# Тестирование с относительным путем
sky test ./utils
```

## Возможности

- **Фреймворк Jest**: Использует Jest для комплексных возможностей тестирования
- **Выборочное тестирование**: Тестирование конкретных папок или всего проекта
- **Очистка консоли**: Автоматическая очистка консоли перед запуском тестов
- **Поддержка TypeScript**: Полная поддержка TypeScript файлов тестов

## Конфигурация

Тесты используют конфигурацию Jest из настроек вашего проекта. Тестовый раннер ищет:
- `*.test.ts` файлы
- `*.test.js` файлы
- `*.spec.ts` файлы
- `*.spec.js` файлы

## Примечания

- Консоль автоматически очищается перед запуском тестов
- Результаты тестов отображаются в терминале с подробным выводом
- Отчеты о покрытии могут быть настроены через конфигурацию Jest
- Поддерживает режим наблюдения и другие опции CLI Jest через конфигурацию
